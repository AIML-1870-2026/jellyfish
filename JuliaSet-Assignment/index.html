Core Features

1. Dual Fractal Modes

Julia Set Mode (Default)

  • Fixed complex parameter c = a + bi
  • Each point z iterated using z² + c
  • User controls the c parameter
  • Default: c = -0.4 + 0.6i (Spiral preset)

Mandelbrot Set Mode

  • Each point c tested by iterating z² + c starting from z = 0
  • Shows the iconic Mandelbrot shape
  • Click any point to generate its Julia set
  • Default view: Centered at (-0.5, 0) with zoom 0.8x

2. Interactive Navigation

Zoom Controls

  • Mouse Wheel: Smooth interpolated zoom (60fps)
  • Zoom Buttons: +/- buttons with animated transitions
  • Reset Button: Return to default view
  • Zoom Indicator: Live display of current zoom level
  • Implementation: Exponential interpolation (15% per frame) for natural feel

Pan Controls

  • Click & Drag: Move around the complex plane
  • Smooth Panning: 60fps throttled for no judder
  • Low-res Preview: During drag for performance
  • High-res Refinement: When drag ends

3. Parameter Controls

Julia Set Parameters

  • Real Component (a): Slider from -2.0 to 2.0, step 0.01
  • Imaginary Component (b): Slider from -2.0 to 2.0, step 0.01
  • Live Preview: Updates as you drag sliders
  • Numeric Display: Shows current values to 2 decimal places

Rendering Parameters

  • Max Iterations: 50–500, default 100 (lower = faster, higher = more detail)
  • Escape Radius: 2–10, default 2 (controls bailout threshold)
  • Throttled Updates: 60fps max to prevent lag

Performance Optimizations

1. Adaptive Quality Rendering

// Low-res mode during interaction (2x2 pixel blocks) const step = state.lowResMode ? 2 : 1; for (let y = 0; y < height; y += step) { for (let x = 0; x < width; x += step) { const iter = calculateFractal(x, y); // Fill block for low-res for (let dy = 0; dy < step; dy++) for (let dx = 0; dx < step; dx++) setPixelColor(x+dx, y+dy, iter); } }

Result: 4x faster rendering during drag/zoom

2. Smooth Interpolated Zoom

// Exponential interpolation for natural feel function smoothZoom() { const diff = targetZoom - currentZoom; if (Math.abs(diff) < 0.001) { currentZoom = targetZoom; state.lowResMode = false; render(); return; } currentZoom += diff * 0.15; // 15% per frame state.zoom = currentZoom; render(); requestAnimationFrame(smoothZoom); }

Result: Buttery smooth 60fps zoom with no judder

3. Optimized Fractal Calculation

  • Pre-computed escape radius squared — Avoid sqrt on every iteration
  • Magnitude squared comparison — Compare zReal² + zImag² directly
  • Reduced variable assignments — Minimize allocations in hot loop
  • Half iterations in low-res mode — Faster interactive feedback

Implementation Details

Julia Set Calculation

function calculateJulia(x, y) { const scale = 4 / state.zoom; const real = (x - width/2) / (width/4) * scale + state.centerX; const imag = (y - height/2) / (height/4) * scale + state.centerY; let zReal = real, zImag = imag, iter = 0; while (zReal*zReal + zImag*zImag <= 4 && iter < maxIter) { const temp = zReal*zReal - zImag*zImag + state.c.real; zImag = 2*zReal*zImag + state.c.imag; zReal = temp; iter++; } return iter; }

Mandelbrot Set Calculation

function calculateMandelbrot(x, y) { const scale = 4 / state.zoom; const cReal = (x - width/2) / (width/4) * scale + state.centerX; const cImag = (y - height/2) / (height/4) * scale + state.centerY; let zReal = 0, zImag = 0, iter = 0; while (zReal*zReal + zImag*zImag <= 4 && iter < maxIter) { const temp = zReal*zReal - zImag*zImag + cReal; zImag = 2*zReal*zImag + cImag; zReal = temp; iter++; } return iter; }

Easing Functions

const easingFunctions = { linear: (t) => t, easeInOut: (t) => t < 0.5 ? 2*t*t : -1 + (4 - 2*t)*t, easeIn: (t) => t*t, easeOut: (t) => t*(2-t), bounce: (t) => { if (t < 0.5) return 8*t*t*t*t; const f = t - 1; return 1 + 8*f*f*f*f; } };

Complex Interpolation

function interpolateComplex(c1, c2, t, easing = 'linear') { const easedT = easingFunctions[easing](t); return { real: c1.real + (c2.real - c1.real) * easedT, imag: c1.imag + (c2.imag - c1.imag) * easedT }; }

Coordinate Transformations

// Pixel to Complex Plane function pixelToComplex(px, py) { const scale = 4 / state.zoom; const real = (px - canvas.width/2) / (canvas.width/4) * scale + state.centerX; const imag = (py - canvas.height/2) / (canvas.height/4) * scale + state.centerY; return { real, imag }; } // Complex Plane to Pixel function complexToPixel(real, imag) { const scale = 4 / state.zoom; const px = (real - state.centerX) * (canvas.width/4) / scale + canvas.width/2; const py = (imag - state.centerY) * (canvas.height/4) / scale + canvas.height/2; return { x: px, y: py }; }

Lessons Learned

Challenge 1: Blank Canvas on Load

Problem: Canvas wouldn't render on initial page load

Solution: Removed validation checks, set explicit fallback dimensions (1200x800), render immediately without delays

"Sometimes the simplest solution is the best. Don't over-engineer."

Challenge 2: Slow Rendering Performance

Problem: Rendering too slow, causing lag during interaction

Solution: Adaptive quality system — low-res (2x2 blocks) during interaction, half iterations during animation, full quality when idle

"Profile before optimizing. The biggest win was adaptive quality, not algorithmic improvements."

Challenge 3: Janky Zoom

Problem: Mouse wheel zoom was jumpy and unpleasant

Solution: Smooth exponential interpolation — currentZoom += (targetZoom - currentZoom) * 0.15

"User experience is about feel, not just functionality."

Challenge 4: Button Responsiveness

Problem: Buttons felt laggy due to over-aggressive debouncing (100–150ms)

Solution: Removed all debouncing from buttons; direct render calls. Throttle only continuous inputs (sliders)

"Different interactions need different strategies."

Challenge 5: Animation Smoothness

Problem: Animations stuttery at 30fps

Solution: Increased frame generation (15x multiplier), reduced iterations 30% during playback, smoother easing

"Smooth animation requires both algorithmic optimization AND adaptive quality."

Best Practices Discovered

Performance

  • Adaptive quality is key — Different quality for different contexts
  • Profile, don't guess — The bottleneck might surprise you
  • Throttle continuously, not discrete — Sliders need throttling, buttons don't
  • Pre-compute when possible — Escape radius squared, not checking every iteration

User Experience

  • Smooth > Fast — 30fps smooth beats 60fps janky
  • Instant feedback — Visual response should be immediate
  • Progressive enhancement — Low-res preview, high-res refinement
  • Clear affordances — Cursor changes, hover states, visual feedback

Code Organization

  • Single file can work — For apps under 5000 lines
  • State centralization — One source of truth
  • Pure functions — Easier to debug and optimize
